# tg-ask-bot

Telegram-бот с AI-ответами на базе OpenRouter и распознаванием изображений через Gemini API.

## Возможности

### Личные чаты
- Отвечает на текстовые сообщения через OpenRouter (DeepSeek V4 Flash по умолчанию)
- Распознаёт изображения через Google Gemini Flash Lite с внутренним рассуждением (`thinkingConfig`) и поиском Google для дополнительного контекста
- Фото понимается в контексте диалога: Gemini описывает изображение, OpenRouter отвечает с учётом истории переписки
- Подпись к фото передаётся вместе с описанием; если подписи нет — бот реагирует на содержимое
- **Fallback на Ollama**: если Gemini заблокировал изображение по политике или упал — запрос уходит на локальную модель (Ollama)
- Если ни Gemini, ни Ollama не смогли — DeepSeek всё равно отвечает в своём стиле
- Персонализация: бот автоматически запоминает ключевые факты о пользователе (город, предпочтения, хобби и т.д.) и использует их в ответах
- Факты извлекаются асинхронно после каждого сообщения, включая косвенные согласия («у меня тоже»), обновления и отзывы («шучу»)
- Хранит до 50 фактов на пользователя с поддержкой обновления и удаления отдельных фактов

### Групповые чаты
- Поддержка групп и супергрупп, включая форум-группы с темами (topics)
- Бот автономно решает, стоит ли вступать в разговор: двухшаговый LLM — сначала decision (последние 5 сообщений), затем ответ (последние 30)
- Whitelist тредов: бот отвечает только там, где включён командами `/botstart` / `/botstop` (только для администраторов)
- Режим «пожиратель» (`/botingest`): бот молча собирает картинки треда в базу, не отвечает и игнорирует текст — удобно для темы-канала с изображениями. Картинки кладутся в устойчивую очередь и обрабатываются фоновым воркером (Gemini параллельно, Ollama строго по одной с circuit breaker и повторами), поэтому поток картинок не роняет локальную модель и переживает рестарт бота. Через 5 минут после обработки последней картинки бот шлёт в чат сводку (сколько обработано, кем, время обработки, топ-теги). Не обработанные после 6 попыток картинки остаются в базе для разбора и ручного повтора (`yarn tsx src/scripts/retryFailedIngest.ts`)
- Поддержка пересланных сообщений (новости, цитаты от каналов и пользователей)
- Метки времени и имена авторов в контексте (часовой пояс Asia/Vladivostok)
- Если кто-то отвечает на сообщение бота (Telegram reply) — decision LLM пропускается, бот отвечает сразу
- Анализ изображений через Gemini во всех группах

### Общее
- **Картинки по ситуации**: бот может прикрепить подходящее изображение к ответу — реакцию, мем или фото; текст и картинка отправляются одним сообщением (caption)
- **Семантический поиск картинок**: все отправленные пользователями изображения сохраняются в базе с описанием, тегами и флагом `is_nsfw`; для поиска используются векторные эмбеддинги (pgvector + Gemini Embedding)
- **Inline-поиск картинок**: в любом чате можно набрать `@имя_бота` + запрос — бот ищет по базе картинок семантически и отдаёт результаты прямо в поле ввода. Пустой запрос показывает случайную подборку. NSFW учитывается по личным настройкам пользователя. Эмбеддинги запросов кэшируются в БД, чтобы не дёргать Gemini на каждое нажатие; одинаковый запрос каждый раз даёт свежую перетасовку из релевантного пула. Требует включения inline-режима у @BotFather (`/setinline`)
- **Импорт с Danbooru** (опционально): фоновый воркер хронологически тянет посты с имиджборда [Danbooru](https://danbooru.donmai.us/) и пополняет ими базу картинок (после чего они доступны в inline-поиске). Грузит не всё подряд, а только «настоявшиеся» популярные работы: пост должен провисеть ≥48ч (за это время формируется оценка) и набрать score ≥5 — пороги выведены из анализа распределения оценок по возрасту. Теги (персонаж, франшиза, автор) идут в описание и эмбеддинг как основные якоря поиска. В чат-хранилище картинки приходят пачками-альбомами (меньше спама), у каждого фото в подписи — ссылка на исходный пост, NSFW отправляется заблюренным (нативный спойлер Telegram). Включается командой `/setdanboorustorage` и требует учётных данных Danbooru
- **NSFW-фильтр**: изображения автоматически классифицируются как 18+ или нет; по умолчанию NSFW-контент скрыт, включается через `/account`
- Сохраняет историю диалога в PostgreSQL (до 50 последних сообщений на пользователя)
- Считает количество запросов на пользователя (только из личных чатов; групповая активность счётчик не увеличивает)
- История сохраняется между перезапусками бота
- Если пользователь пишет пока бот обрабатывает запрос — получает забавный ответ вместо зависания
- Поддержка прокси
- HTML-форматирование ответов с fallback на plain text при невалидной разметке
- Логирование через pino с ротацией по дням

## Команды

| Команда | Описание | Где работает |
|---------|----------|-------------|
| `/start` | Приветствие | Везде |
| `/help` | Список команд | Везде |
| `/botstart` | Включить бота в текущем треде (только для админов) | Группы |
| `/botingest` | Режим «пожиратель»: молча собирать картинки в базу (только для админов) | Группы |
| `/botstop` | Отключить бота в текущем треде (только для админов) | Группы |
| `/clear` | Очистить историю диалога | Только личные чаты |
| `/facts` | Управлять сохранёнными фактами | Только личные чаты |
| `/account` | Профиль, статистика и настройки (включая NSFW) | Только личные чаты |
| `/setdanboorustorage [start_id]` | Назначить текущий чат/тему хранилищем для импорта с Danbooru (только для админов) | Группы |

## Установка

```bash
git clone https://github.com/MKRED/tg-ask-bot.git
cd tg-ask-bot
yarn install
```

## Настройка

Создай файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполни переменные:

```env
BOT_TOKEN=            # токен от @BotFather
OPENROUTER_API_KEY=   # ключ на openrouter.ai
GEMINI_API_KEY=       # ключ на aistudio.google.com
GEMINI_API_KEY_FREE=  # опционально: бесплатный ключ, используется только скриптом reembedImages.ts
DATABASE_URL=         # postgres://user:password@host:5432/dbname
PROXY_URL=            # опционально: http://host:port
OPENROUTER_MODEL=     # опционально, по умолчанию deepseek/deepseek-v4-flash
OLLAMA_URL=           # опционально, по умолчанию http://localhost:11434
OLLAMA_VISION_MODEL=  # опционально, по умолчанию gemma4-vision
DANBOORU_LOGIN=       # опционально: логин на danbooru.donmai.us (для импорта картинок)
DANBOORU_API_KEY=     # опционально: API-ключ Danbooru (профиль → API Key)
```

Импорт с Danbooru включается только если заданы обе переменные `DANBOORU_LOGIN` и `DANBOORU_API_KEY`. После старта бота назначь чат-хранилище командой `/setdanboorustorage` (в нужной группе/теме) — туда воркер будет загружать картинки, чтобы получить Telegram `file_id` для inline-поиска.

### Настройка бота для групп

Перед добавлением в группу обязательно отключи Privacy Mode в @BotFather:
- `/mybots` → выбрать бота → Bot Settings → Group Privacy → **Disable**

Без этого бот видит в группах только команды и ответы на свои сообщения.

После изменения настройки удали бота из всех тестовых групп и добавь заново.

### Включение inline-поиска картинок

Чтобы заработал inline-режим (`@имя_бота` + запрос в любом чате):
- `/setinline` в @BotFather → выбрать бота → задать placeholder (например `найти картинку…`)

Без этого Telegram не присылает боту inline-запросы вообще.

## База данных

Требуется PostgreSQL 17+ с расширением [pgvector](https://github.com/pgvector/pgvector) (для семантического поиска картинок).

```bash
# Применить миграции
yarn drizzle-kit migrate

# Сгенерировать новую миграцию после изменений схемы
yarn drizzle-kit generate
```

## Запуск

```bash
# Режим разработки (с hot reload)
yarn dev

# Продакшн
yarn build
yarn start
```

## Docker

Нужен образ с поддержкой pgvector, например `pgvector/pgvector:pg17`:

```bash
docker build -t tg-ask-bot .
docker run --env-file .env tg-ask-bot
```

## Стек

- [grammY](https://grammy.dev/) — Telegram Bot framework
- [OpenRouter](https://openrouter.ai/) — LLM API (текст, decision)
- [Google Gemini](https://aistudio.google.com/) — Vision API (изображения, `gemini-3.1-flash-lite`) + Embedding API (`gemini-embedding-2`, мультимодальная: текст и картинки в одном векторном пространстве)
- [Ollama](https://ollama.com/) — локальная Vision модель, fallback при блокировке Gemini
- [Drizzle ORM](https://orm.drizzle.team/) + PostgreSQL + pgvector — хранение истории, фактов, изображений, групповых буферов и векторный поиск
- [Danbooru API](https://danbooru.donmai.us/wiki_pages/help:api) — опциональный источник картинок для пополнения базы
- [pino](https://getpino.io/) — логирование
- TypeScript + tsx
